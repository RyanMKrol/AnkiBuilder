## Overview

Render the human review "audit table" so a build can be eyeballed against the source before it's
trusted — the review gate the baseline skill treats as mandatory.

## Do

Add `src/audit/` that, given a `cards.json`, produces a review table with one row per card
(English · Target · Pronunciation · Category · Image? · Audio?), grouped/sorted sensibly, followed
by totals.

## Done when

- `renderAuditTable(cards)` returns a table string with those columns, using ✓/· flags for image
  and audio presence, followed by totals (total cards, count with image, count with audio).
- Column alignment stays readable when target text uses wide/non-Latin scripts.
- `expectsTest`: `test/audit/` asserts that for a small sample the rendered output contains each
  card's English and target text, the correct ✓/· flags for image/audio, and correct totals.
