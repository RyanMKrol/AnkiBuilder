# T008: Audit-table renderer for review gates

**Date:** 2026-07-13

## What was done

Implemented `src/audit/index.js` with `renderAuditTable(cards)` function that renders a human-readable audit table for reviewing Anki card builds before they're trusted.

### Implementation details

- **Function:** `renderAuditTable(cards)` takes a cards object with an `items` array and returns a formatted string table
- **Columns:** English, Target, Pronunciation, Category, Image?, Audio?
- **Flags:** Uses ✓ for presence, · (middle dot) for absence of image/audio
- **Sorting:** Groups by category, then sorts alphabetically by English text within each category
- **Totals row:** Shows total card count, count with image, count with audio
- **Wide character support:** Implements `displayWidth()` helper that accounts for CJK characters (Chinese, Japanese, Korean, etc.) which display as 2-column width, enabling proper alignment even with mixed scripts
- **Column width calculation:** Dynamically adjusts based on content with sensible min/max bounds

### Test coverage

Created comprehensive test suite in `test/audit/renderAuditTable.test.js` with 13 test cases asserting:
- Presence of all required columns (English, Target, Pronunciation, Category, Image?, Audio?)
- Correct ✓/· flags for image and audio presence/absence
- Accurate image and audio count totals
- Proper grouping and sorting by category then English text
- Wide/non-Latin script handling (CJK characters display correctly and tables remain aligned)
- Edge cases: empty cards, missing items property, null/undefined input, accurate total counts
- All card data is preserved in the output
