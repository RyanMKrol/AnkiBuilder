## Overview

Establish the pipeline's data contracts and filesystem conventions. Every later stage
(corpus assembly, translation, audio, deck build) reads and writes these, so this is the spine
the rest of the project binds to.

## Do

Create the data model under `src/model/`: JSON Schemas plus a Node module that loads and validates
the two pipeline artifacts (`corpus.json`, `cards.json`), and helpers for the run-directory layout
and the state home. This project is a Node CLI/library; keep library modules under `src/` and tests
under `test/`.

## Done when

- A `corpus.json` schema is defined:
  `{ meta: { targetLanguage: string, sourceType: "template" | "epub" | "manual" }, items: [ { id: string, english: string, category: string, notes?: string } ] }`.
- A `cards.json` schema is defined: a card is a corpus item plus
  `{ target: string, pronunciation: string, hint?: string, image?: string, audio?: string }`.
- `src/model/` exports `validateCorpus(obj)` and `validateCards(obj)` that reject malformed input
  with a clear error and accept valid input, plus `stateHome()` (reads `ANKI_BUILDER_HOME`, defaults
  to `~/.anki-builder`, and MUST resolve outside the repo — never inside git) and a `runPaths(runDir)`
  helper returning the conventional paths (`corpus.json`, `cards.json`, `audio/`, `deck.apkg`).
- `expectsTest`: `test/model/` has tests asserting: a valid corpus and a valid cards object pass
  validation; a corpus item missing `english` or `category` fails; a cards object missing `target`
  fails; `stateHome()` honours `ANKI_BUILDER_HOME` when set and falls back to the default when unset.
