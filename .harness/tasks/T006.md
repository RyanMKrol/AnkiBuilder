## Overview

Generate spoken audio for each card's target-language term via ElevenLabs TTS, cached so re-runs
cost no quota. Ports the proven pattern from the baseline `jbp-lesson-deck` skill, generalized off
Japanese.

## Do

Add `src/audio/` that reads a `cards.json`, generates one MP3 per unique `target` term via
ElevenLabs into a voice-specific cache dir under the state home, and annotates each card with its
audio filename. Put the HTTP call behind an **injectable interface**; read the API key from the
`ELEVENLABS_API_KEY` environment variable only.

## Done when

- `generateAudio(cards, { voiceId, fetchTts })` writes one MP3 per unique `target` term, named by a
  stable hash of the term (same term → same filename across runs), into
  `<stateHome>/audio/<voiceId>/`, and returns the cards annotated with their `audio` filenames.
- It is idempotent/cached: a term whose MP3 already exists is skipped and triggers no call.
- `ELEVENLABS_API_KEY` is read from env only and never written to disk; a missing key is a clear
  error.
- `expectsTest`: `test/audio/` injects a mock `fetchTts` (NO live HTTP) and asserts: an MP3 is
  written per unique term; a second run with files present makes zero calls (cache hit); the same
  term yields the same filename; a missing API key errors. Tests write only to a scratch/temp dir.
