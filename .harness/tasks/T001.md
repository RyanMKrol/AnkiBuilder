## Overview

🔒 **needs-human.** The one-time foundational scaffold — it *defines* the project-wide
conventions (formatter, linter, CI) that every later task's scope gate depends on. It is
intentionally human-owned: a bootstrap task is inherently cross-cutting, so a narrow per-file
scope gate is the wrong tool for it (a repo-wide formatter's blast radius collides with a tight
file list — see the interrupted first run, where `prettier --check .` drove the builder to
reformat the out-of-scope `CLAUDE.md`). Do it interactively with full judgment, then mark it done.

## Do (human, interactive)

- Create the Node skeleton: `package.json` (name `anki-builder`, `"type": "module"`, `scripts` for
  `format:check` / `lint` / `test` / `build`), `src/index.js` (trivial entrypoint),
  `test/smoke.test.js` (one trivial `node:test`). `.env.example` already exists — keep it.
- Configure Prettier + ESLint. **Crucial lesson from the interrupted run:** keep the formatter's
  blast radius aligned with the code, so it never rewrites hand-written prose that a scoped task
  can't touch. Do ONE of:
  - scope the format scripts to code dirs — `prettier --check src test` (and `--write src test`), or
  - ship a `.prettierignore` excluding the committed prose + harness (`CLAUDE.md`, `.harness/**`,
    and optionally `README.md`).
- Run Prettier once so the repo is clean and *stays* green under `format:check`.
- Confirm `npm ci`, `npm run format:check`, `npm run lint`, `npm test`, `npm run build` all pass
  locally, and that CI (`.github/workflows/ci.yml`) is green on `main`.

## Done when

- All Definition-of-Done commands pass locally and CI is green on `main`.
- The repo is Prettier-clean with a committed config whose ignore/scope excludes the hand-written
  prose (`CLAUDE.md`, `.harness/**`), so no future scoped task trips the whole-repo-formatter gate.
- Marked done via `.harness/scripts/mark-done.sh T001` (this is a `needs-human` task; the loop never
  builds it — completing it is the interactive session + the mark-done overlay).
