## Overview

Author the thin Claude skill that conversationally drives the pipeline and handles ambiguity for the
user. This is the product's user-facing form — scripts do the work, the skill coordinates.

## Do

Write `skills/build/SKILL.md` — a Claude skill that: asks whether the user has an EPUB or wants a
bundled template (and which one and which target language), assembles the corpus, presents the audit
table at each review gate, then runs the translate → audio → build steps via the T009 CLI, ending by
handing the `.apkg` to the user to import and verify. Keep the Claude layer thin — it coordinates
and resolves ambiguity; the scripts carry the logic.

## Done when

- `skills/build/SKILL.md` exists with valid frontmatter (`name` + `description`) and documents the
  full flow: the "EPUB or template?" entry decision, corpus assembly + its review gate, translation
  via `claude -p` + the audit-table review, ElevenLabs audio (voice choice; `ELEVENLABS_API_KEY` via
  env only — noting the user can put it in a gitignored `.env` copied from `.env.example`, which the
  CLI loads automatically), deck build, and the final human import/verify in Anki.
- It invokes the **actual** CLI commands/scripts from T009 (not invented ones) and references the
  state-home / env-key conventions from T002/T006, hard-coding nothing about file locations.
- No automated test (this is a prose skill; it is validated end-to-end by T011).
