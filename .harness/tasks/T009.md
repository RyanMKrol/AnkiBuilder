## Overview

Provide a script-level orchestrator so the whole pipeline can run without Claude, letting the skill
layer (T010) sit thinly on top. Keeps the Claude layer genuinely thin — the scripts do the work.

## Do

Add `src/cli/` and wire a `bin` entry in `package.json` exposing subcommands that drive the stages
over a run directory: assemble a corpus (from a bundled template or an EPUB), translate, generate
audio, and build the deck — each reading the prior artifact and writing the next so a run is
resumable. Reuse the stage modules from T003–T008; do not duplicate their logic.

The CLI must also make local secrets convenient without ever writing them to disk: when a `.env`
file is present in the current working directory, load it into `process.env` at startup (use Node's
native `--env-file=.env`, e.g. via the `bin` shebang / npm script, so no new dependency is added),
so `ELEVENLABS_API_KEY` and other vars documented in `.env.example` are available to the audio stage
without the user exporting them by hand. The `.env` file is read-only input and stays gitignored;
the CLI never writes it.

## Done when

- The CLI exposes commands to: assemble a corpus (choosing a template via T003 or an EPUB via T004),
  run translation (T005), run audio (T006), and build the deck (T007) — reading/writing
  `corpus.json` → `cards.json` → `deck.apkg` in a run directory (paths per T002's `runPaths`).
- Re-running a stage whose output already exists reuses it rather than redoing the work (resumable).
- `package.json` gains a `bin` mapping to the CLI entrypoint.
- When a `.env` file is present in the working directory, the CLI loads its variables into the
  environment at startup (via Node's native `--env-file`); when absent, it runs fine using whatever
  is already exported. The `.env` is never written to.
- `expectsTest`: `test/cli/` asserts command dispatch and stage wiring using injected/mocked stage
  dependencies (NO live `claude`, ElevenLabs, or network), including that a stage is skipped when its
  output artifact already exists, and that a variable set only in a scratch `.env` file is loaded
  into the environment when the CLI runs.
