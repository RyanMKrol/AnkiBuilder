# T006: ElevenLabs audio stage (cached, idempotent)

## 2026-07-13

### Implementation

Implemented `generateAudio` function in `src/audio/index.js` with the following characteristics:

1. **Function signature**: `generateAudio(cards, { voiceId, fetchTts, stateHomeDir })`
   - `voiceId`: Required voice ID for ElevenLabs
   - `fetchTts`: Injectable HTTP function (no live calls in tests)
   - `stateHomeDir`: Optional state directory override for testing

2. **Core behavior**:
   - Reads `ELEVENLABS_API_KEY` from environment only (never written to disk)
   - Generates one MP3 per unique `target` term using stable SHA256-based hashing
   - Stores files in `<stateHome>/audio/<voiceId>/`
   - Returns cards annotated with `audio` filenames

3. **Idempotency/Caching**:
   - Checks if MP3 already exists before calling `fetchTts`
   - Second run with cached files makes zero API calls
   - Same term always maps to same filename across runs

4. **Error handling**:
   - Throws clear error if `ELEVENLABS_API_KEY` missing
   - Throws error if required parameters (`voiceId`, `fetchTts`) not provided

### Tests

Created comprehensive test suite in `test/audio/index.test.js` with 11 tests covering:

1. **Writes one MP3 per unique term** into voice-specific cache directory

All tests use scratch/temp directories (no production state mutation).
