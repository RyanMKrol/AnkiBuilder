# T004 — EPUB corpus extractor

Date: 2026-07-13

## What I did

- Added `src/corpus/epub.js` exporting `extractEpub(epubPath, { targetLanguage })`.
  - No zip/epub library exists in `package.json` (only devDependencies: eslint, prettier),
    and `package.json` is out of this task's scope, so the ZIP container (local file
    headers, central directory, end-of-central-directory record) is parsed by hand using
    only Node built-ins (`fs`, `zlib.inflateRawSync` for deflate entries, raw copy for
    stored entries).
  - Content documents (`*.xhtml`/`*.html`/`*.htm`) are scanned for block-level text
    (`<p>`, `<li>`, `<dt>`, `<dd>`, `<td>`), with tags stripped and common HTML entities
    decoded.
  - Each text block is parsed as a candidate: if it contains a delimiter (`—`, `–`, `:`,
    `-`, `--`) it's split into an English term plus a translation, which is captured in
    the item's `notes` field; otherwise it's kept as a standalone English candidate term
    with a default `category: "General"`.
  - Duplicate English terms (case-insensitive) are deduped.
  - The result is validated with the existing `validateCorpus` from `src/model/index.js`
    before being returned, and always carries `meta.sourceType: "epub"`.
  - Extraction is hermetic: only `fs.readFileSync(epubPath)` is used to read input; no
    network calls, no hard-coded paths.
