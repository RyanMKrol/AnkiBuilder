## Overview

Let users seed a corpus from a real reference book by extracting candidate terms from a
user-supplied EPUB. Output is *candidates* for human review, not a finished corpus.

## Do

Add `src/corpus/epub.js` that unzips a user-supplied EPUB and extracts candidate vocabulary
terms/phrases (plus any translations that appear alongside them in the source) into a schema-valid
`corpus.json` (`meta.sourceType: "epub"`), with items flagged as candidates for later human review.
Nothing about the EPUB location is hard-coded — the path is always passed in.

## Do not

Do not attempt per-book/per-language heuristics beyond generic term extraction; anything smarter is
a later phase.

## Done when

- `extractEpub(epubPath, { targetLanguage })` returns a `corpus.json` object whose items each carry
  `english` and a `category` (use a sensible default category when the source has none). When the
  source text pairs an English term with a target-language translation, capture that translation in
  the item's `notes` so the translation stage (T005) can reuse rather than regenerate it.
- Extraction is hermetic: it reads only the given file and makes no network calls.
- The returned object validates against T002's `corpus.json` schema.
- `expectsTest`: `test/corpus/epub.test.js` uses a small fixture EPUB under `test/fixtures/` (a
  minimal zip with a couple of xhtml pages — construct it in-test or commit it) and asserts the
  extractor returns the expected candidate items and that the result validates against the schema.
  No network and no hard-coded external paths.
