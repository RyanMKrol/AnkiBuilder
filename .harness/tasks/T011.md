## Overview

Prove the whole tool works by building a real travel-onboarding deck end to end and checking it
inside the Anki desktop app. This is a human acceptance step — it needs real credentials and a human
to eyeball and listen.

## Do

Following `skills/build/SKILL.md`, build a travel deck for one target language from the bundled
travel template, generate audio with a real `ELEVENLABS_API_KEY`, produce the `.apkg`, import it
into Anki, and verify the result.

## Done when

- A `.apkg` is produced from the travel template for a chosen target language and imports cleanly
  into Anki.
- Cards show correct target text + pronunciation + English + category; both the Recognition and
  Production templates render correctly; audio plays on the cards that have it.
- Any defects found are captured as follow-up backlog tasks or ideas (do not fix them here).
