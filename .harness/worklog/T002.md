# T002: Pipeline data contracts + run-dir/state-home conventions

## 2026-07-13

### What was done

Implemented the data model layer for the pipeline with JSON schemas, validation functions, and path helpers.

**Implemented files:**
- `src/model/index.js` — Core module with:
  - `CORPUS_SCHEMA`: JSON schema for corpus artifacts with meta (targetLanguage, sourceType) and items array
  - `CARDS_SCHEMA`: JSON schema for enriched cards (corpus items + target language fields)
  - `validateCorpus(obj)`: Validates corpus objects against schema; throws with clear errors for missing/invalid fields
  - `validateCards(obj)`: Validates cards objects against schema; throws with clear errors for missing/invalid fields
  - `stateHome()`: Returns resolved path from `ANKI_BUILDER_HOME` env var or defaults to `~/.anki-builder`; always resolves outside repo
  - `runPaths(runDir)`: Returns object with conventional paths: `{ corpus, cards, audio, deck }`

- `test/model/index.test.js` — Comprehensive test suite covering:
  - Valid corpus/cards objects pass validation
  - Missing required fields (english, category, target, pronunciation) fail with clear errors
  - Invalid enum values (sourceType) fail validation
  - Optional fields (notes, hint, image, audio) are accepted
  - `stateHome()` honours `ANKI_BUILDER_HOME` when set, falls back to default when unset
  - `stateHome()` returns fully resolved paths (no `~`)
  - `runPaths()` returns absolute paths
