## Overview

Assemble the final importable Anki `.apkg` from the translated cards (and any audio). This is the
highest-risk, highest-value stage — the actual deck artifact.

## Do

Add `src/deck/` that, given a `cards.json` (and optionally the audio cache dir), builds a valid
`.apkg` with a two-template note model — **Recognition** (target → English) and **Production**
(English → target) — and fields Target, Pronunciation, English, Category, Hint, Image, Audio,
attaching each card's audio when the file exists. Choose the `.apkg` tooling that works best in Node
(a genanki-style library, or building the SQLite `collection.anki2` + media zip directly). If — and
only if — no viable Node path exists, stop and flag it for a follow-up rather than silently switching
the project's stack.

## Done when

- `buildDeck(cards, { outPath, audioDir })` writes a `.apkg` at `outPath` that Anki can import: a
  zip containing a `collection.anki2` SQLite DB with one note per card, two cards per note (the two
  templates), and a `media` mapping for any attached audio.
- Audio is attached only when its file exists; a card with no audio still produces valid cards.
- `expectsTest`: `test/deck/` asserts the produced `.apkg` is a valid zip, contains
  `collection.anki2` with the expected note count and both templates, that field values round-trip,
  and that referenced media entries resolve to files in the package. Tests write to a scratch/temp
  output path, never a real user deck or collection.
