## Overview

Produce target-language translations for corpus items that lack them, using the local `claude -p`
CLI (not an API). This is where the target language actually enters the cards.

## Do

Add `src/translate/` that takes a `corpus.json` and fills each item's `target`, `pronunciation`, and
optional `hint` by invoking the local `claude -p` binary with a structured prompt, producing a
`cards.json`. Put the `claude -p` invocation behind an **injectable runner** so tests never call the
real binary.

## Done when

- `translateCorpus(corpus, { runClaude })` returns a schema-valid `cards.json` (validated with
  T002's `validateCards`): each item gains `target` + `pronunciation`, and a `hint` when the model
  supplies one. `runClaude` defaults to a real `claude -p` subprocess wrapper and can be injected.
- Items that already carry a translation (e.g. an EPUB item whose `notes` captured a source
  translation) are reused/verified, not blindly regenerated.
- The prompt asks the model for structured JSON; the parser tolerates and **reports** malformed
  model output as an error rather than emitting an invalid card.
- `expectsTest`: `test/translate/` injects a mock `runClaude` (NO real `claude` call) and asserts:
  untranslated items get `target`+`pronunciation` merged into valid cards; an already-translated
  item is preserved; malformed model output is surfaced as an error, not written as a bad card.
