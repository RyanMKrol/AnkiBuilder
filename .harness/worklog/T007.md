# T007 — .apkg deck builder (two-template model)

2026-07-13

## What I did

Added `src/deck/`:

- `zip.js` — minimal ZIP writer (deflate + CRC32, hand-rolled since no dependency could be
  added: `package.json` is outside this task's scope). Builds local file headers, a central
  directory, and an EOCD record from `{ name, data }` entries.
- `collection.js` — builds the raw bytes of a legacy-schema (`ver: 11`) `collection.anki2`
  SQLite database using Node's built-in `node:sqlite` (`DatabaseSync`), avoiding a new
  dependency entirely. One model (`AnkiBuilder`) with two templates — **Recognition**
  (Target → English) and **Production** (English → Target) — and seven fields: Target,
  Pronunciation, English, Category, Hint, Image, Audio. One note per card, two cards per
  note (one per template ordinal). `Image`/`Audio` field values are rendered as
  `<img src="...">` / `[sound:...]` only when the corresponding filename is present.
  `DatabaseSync` has no in-memory serialize, so the DB is built in a temp file and read back
  as bytes, then the temp dir is removed.
- `index.js` — `buildDeck(cards, { outPath, audioDir, deckName, now })`: resolves each card's
  audio against `audioDir` (existsSync check), only attaches audio that actually exists on
  disk, assembles the `media` mapping (`{"<index>": "<filename>"}`) genanki/Anki expects,
  writes `collection.anki2` + `media` + numbered media entries into the zip via `zip.js`, and
  writes the result to `outPath` (creating parent dirs as needed).

Chose to build the SQLite + zip directly rather than pull in a genanki-style npm package:
`package.json`/`package-lock.json` are outside this task's scope (`src/deck/**`,
`test/deck/**` only), so adding any new dependency was not an option without tripping the
scope gate. Node 22's built-in `node:sqlite` (experimental but functional) made a
dependency-free implementation viable, so no follow-up flag was needed.

## Tests (`test/deck/index.test.js`)

- Asserts the produced `.apkg` is a valid zip (hand-parses the EOCD + central directory) and
  contains `collection.anki2`, `media`, and a numbered media entry for attached audio.
- Extracts `collection.anki2` from the zip, opens it with `node:sqlite`, and asserts: note
  count, 2 cards per note (both template ordinals present), the model's template names are
  `["Recognition", "Production"]`, field order matches the required 7 fields, and field
  values round-trip correctly (Target/Pronunciation/English/Category/Hint).
- Asserts a card with no audio (or an audio filename that doesn't exist on disk) still
  produces valid cards and no dangling/missing media entry.

All tests write to `fs.mkdtemp`-created scratch directories, never a real user deck or
collection.
